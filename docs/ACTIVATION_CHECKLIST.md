# قائمة بيانات التفعيل المطلوبة من المالك

لا ترسل الأسرار في الدردشة. جهّزها وأدخلها لاحقًا في `.env` على الخادم.

## هوية المتجر

- اسم المتجر.
- معرف الدعم في تيليجرام.
- Telegram ID لكل أدمن.
- سياسة الضمان والاستبدال والرد.
- شروط الاستخدام والخصوصية.

## تيليجرام والاستضافة

- توكن جديد من BotFather.
- خادم Linux ونطاق HTTPS.
- PostgreSQL وRedis ونسخ احتياطي.
- عنوان `PUBLIC_BASE_URL` ثابت.

## المورد

- ملف Swagger/OpenAPI الكامل أو صور جميع أجزاء إنشاء واستعلام الطلب.
- معرّف منتج Gemini Pro الصحيح والمدة المتاحة.
- سعر التكلفة والحد الأدنى لرصيد المورد.
- مثال طلب واستجابة من sandbox مع إخفاء المفتاح.
- تأكيد مكتوب أن إعادة البيع مسموحة.
- تأكيد دعم مفتاح منع تكرار أو طريقة الاستعلام بعد timeout.

## Binance Pay

- حساب Binance Pay Merchant مفعل في بلد النشاط.
- API identity key وsecret من بوابة التاجر.
- نطاق webhook مسموح في بوابة التاجر.
- عملة التسوية المتاحة (USDT أو USDC بحسب الحساب والمنطقة).
- تجربة دفع صغيرة ثم إعادة إرسال webhook للتأكد من منع التكرار.

## الكريمي

- حساب/نقطة تاجر وخدمة API أو حاسب مفعلة.
- وثائق المصادقة وsandbox وwebhook من البنك.
- العملات وحدود العملية والرسوم.
- إذا لم تُفتح API: رقم الحساب/المحفظة وتعليمات التحويل للمراجعة اليدوية.

## جيب

- عقد تاجر ووثائق API رسمية إن كانت متاحة لك.
- وإلا: رقم المحفظة وتعليمات التحويل وحدود الشحن للمراجعة اليدوية.

## كل خدمة

- الاسم والمدة والوصف.
- سعر البيع والتكلفة.
- معرّف المنتج لدى المورد.
- البيانات المطلوبة من العميل، من دون كلمة مرور أو OTP.
- وقت التسليم والضمان وشروط الرد.
- هل التنفيذ يدوي أم آلي؟
