# حالة المشروع

## مكتمل في النسخة 0.3.0

- واجهة عميل عربية للمتجر والمحفظة والطلبات والشحن والدعم.
- لوحة أدمن للأسعار والخدمات والأقسام والأرصدة وطرق الدفع والمراجعة والتسليم والرد.
- دفتر قيود مالي ومنع تكرار الخصم والشحن.
- تنفيذ يدوي وآلي مع عزل النتائج غير المؤكدة.
- Binance Pay Merchant مع طلب الدفع وفحص webhook الموقّع.
- شحن يدوي آمن لجيب والكريمي.
- موصل VenteBot كامل مع جلب المنتجات والصور والمخزون والضمان والتكلفة.
- تسعير المورد بالنسبة والحد الأدنى للربح، مع قفل السعر والتوفر اليدوي لكل خدمة.
- متابعة طلبات المورد المعلقة دون تكرار الطلب أو الخصم.
- مزامنة يدوية من لوحة الأدمن ومزامنة خلفية دورية.
- PostgreSQL وRedis وDocker وAlembic.
- 20 اختبارًا آليًا، وفحص تنسيق وجودة، وفحص تطابق migrations.

## متوقف على بيانات خارجية

- مفتاح VenteBot جديد ورصيد مورد كافٍ وتجربة شراء صغيرة في البيئة الحقيقية.
- مفاتيح Binance Pay Merchant وحساب تاجر صالح وبيئة اختبار.
- عقد ووثائق API من بنك الكريمي لتحويل المسار اليدوي إلى تلقائي.
- وثائق API تجارية من جيب إن كانت متاحة؛ وإلا يبقى الاعتماد يدويًا.
- نطاق HTTPS وخادم الإنتاج.
- أسعار المنتجات وشروط الضمان والرد ومعلومات الدعم.

## ممنوع قبل الإطلاق

- تفعيل خدمة آلية قبل تجربة المورد بحساب ومبلغ اختبار.
- استخدام مفتاح المورد الظاهر في لقطة الشاشة؛ يجب تدويره.
- طلب كلمة مرور Google أو OTP من العميل.
- تفعيل محفظة محلية قبل ضبط رقم الحساب وسعر الصرف.
- فتح PostgreSQL أو Redis مباشرة على الإنترنت.
